# 策划给开发的文档

这份文档以后固定用于记录：
- 策划确认后的开发补充需求
- 当前版本尚未接完、但后续必须补齐的系统能力
- 希望主程优先做成配置化的运行时与工具支持

这份文档是策划写给开发看的固定文档。
如果是开发反向告诉策划“当前哪些字段已接入、怎么配、有什么约束”，统一看：
`Docs/策划配置总说明.md`

## 当前版本背景

- 当前主流程关卡数按 `StageNode.csv` 配置读取
- Boss 关卡位置按 `StageNode.csv` 配置
- 最终首领关由策划在 `StageNode.csv` 中配置
- 普通怪 / 精英怪保留五行标签
- 战斗公式由 `BattleFormula.csv` 配置读取
- 五行与圣元素关系由 `ElementRelation.csv` 配置读取
- 章节 Boss 统一使用 `圣元素标签`
- `圣元素` 不参与五行增减伤、减伤、克制判定
- Boss 机制风格仍分别对应 `金 / 木 / 水 / 火 / 土`
- 当前数值和文案尽量希望继续走配置给策划维护

## 一、技能效果系统需要补成可配置框架

### 当前问题

目前技能系统更偏“单效果、字符串分支”结构，已经不够支撑后续章节怪、Boss 机制、事件 buff、持续效果扩展。

### 需要补的能力

- `Buff / Debuff` 运行时结构
- 持续回合数与触发时机
- 叠层规则
- 覆盖规则
- 驱散 / 免疫 / 抵抗规则
- 护盾独立承载
- DOT / HOT / 易伤 / 破甲 / 回能 / 控制 / 抗性变化 的统一效果承载
- 支持“一个技能带多个效果”

### 策划希望的结果

- 后续技能不要只靠一个 `EffectType` 硬分支
- 关键效果参数尽量能走配置
- 持续时间、倍率、叠层上限不要散落写死在代码里
- 技能本体与技能效果分表维护，建议固定为 `Skill.csv + SkillEffect.csv`

## 二、战斗公式需要配置化

### 当前问题

关键战斗常量和公式如果继续写死，后面 `100` 关长线、五行怪特征、Boss 机制、事件 buff 都会越来越难调。

### 需要支持的配置项

- 基础伤害系数
- 防御折算方式
- 技能倍率系数
- 治疗系数
- 护盾系数
- 暴击相关系数
- 易伤 / 破甲 / 受疗加成等乘区或加区
- 五行克制系数上限
- `圣元素` 豁免规则
- Boss 特殊系数上限

### 策划希望的结果

- 做一个统一的战斗公式配置源
- 后续策划主要调配置，不再频繁找主程改公式常量

## 三、五行关系与圣元素需要正式接入

### 当前目标

- 普通怪 / 精英怪保留 `金木水火土`
- 五行怪的特征要真实生效，不只是名字和说明
- Boss 统一使用 `圣元素`
- `圣元素` 对普通五行默认中性
- Boss 继承章节五行风格，但不继承五行克制关系

### 需要主程补的能力

- 元素关系统一判定入口
- 普通元素之间的克制配置化
- `圣元素` 中性判定配置化
- 后续支持扩展特殊标签，例如：`魔 / 仙 / 混沌`

### 策划目的

- 保留五行怪辨识度
- 避免某个主角被某章 Boss 克得过死
- 后续如果要做特殊首领，不需要再重写整套元素判定

## 四、敌人与 Boss 阵容要走配置

### 当前问题

如果关卡已经扩成 `100` 关，但战斗阵容仍然写死，章节体验和五行差异会非常弱。

### 需要支持的配置能力

- 普通战 / 精英战 / Boss 敌人池配置
- 按章节切敌组
- 按关卡段切敌组
- 按节点类型切敌组
- Boss 单独配置
- 支持章节内不同五行怪混编
- Boss 单独指定 `圣元素标签 + 章节机制模板`

### 策划目的

- 章节感真正落到玩法上
- 五行怪不是纯文案差异
- Boss 是章节玩法的收束，不只是更大数值怪

### 当前接入约定

- 敌方编组统一走 `EnemyEncounter.csv`
- 运行时按 `StageFrom / StageTo + NodeType` 选编组
- Boss 可在编组表里单独指定 `OverrideElement = Holy`

## 五、怪物特征要落成真实机制

### 当前策划定义

- 金：压制、破甲、点杀
- 木：回复、缠绕、拖战
- 水：减速、冻结、护盾、节奏干扰
- 火：爆发、AOE、灼烧
- 土：高防、护盾、反击、持久战

### 需要主程补的能力

- 至少让每种特征有一到两个真实可运行的效果承载
- 这些效果尽量走统一配置结构，不要每个怪都手写特例逻辑

### 策划目的

- 玩家能明显感受到“五行怪打起来不一样”
- 章节差异来自机制，而不是只来自数值高低

## 六、事件收益与战斗收益继续拆开配置

### 当前目标

- 战斗收益、非战斗收益、事件消耗、事件奖励、功法奖励档位，尽量分开配置
- 长线关卡后，中后期收益节奏不能只靠代码公式硬推

### 需要主程支持

- 战斗收益支持按关卡或章节配置
- 非战斗节点收益支持按 `Profile` 配置
- 固定服务点与随机机缘分开结算
- 事件冷却和服务点机制尽量配置化

## 七、配置校验工具继续补强

### 当前建议增加的校验项

- `StageNode.csv` 是否完整覆盖当前策划配置的关卡范围
- Boss 关卡是否都已正确配置到 `StageNode.csv`
- `EventProfile` 是否都能在 `EventOption.csv` 找到 `1 / 2 / 3`
- `Localization key` 是否存在
- 技能效果字段是否完整
- 元素标签是否合法
- `圣元素 Boss` 是否误吃五行克制
- 敌人池 / Boss 配置是否缺档

## 八、配置优先级约定

以后新增功能时，优先级建议是：
1. 先定义策划字段和配置结构
2. 再接运行时读取
3. 再补编辑器导表与校验
4. 最后保留必要默认值

### 目标

- 数值和文字尽可能走配置给策划维护
- 主程主要负责：框架、读取、校验、运行时结算
- 策划主要负责：填表、调数、改文案、迭代节奏
