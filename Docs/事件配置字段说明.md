# 事件配置字段说明

## 目的

这份文档对应两张表：
- `Docs/StageNode.csv`
- `Docs/EventOption.csv`

现在地图玩法已经拆成两层配置：
1. 先用 `StageNode.csv` 决定每一关是什么节点、走哪种事件档位。
2. 再用 `EventOption.csv` 决定这个事件档位下的三个选项分别给什么、消耗什么。

也就是说，后面如果策划想把“资源交易点”从第 4 关改到第 6 关，或者把“悟道静室”的第一项从 `经验+50` 改成 `经验+30`，都可以直接改表，不需要改代码。

## StageNode.csv

### 字段说明

`Stage`
- 关卡编号。
- 当前版本默认从 `1` 开始递增。

`NodeType`
- 节点类型。
- 当前支持：`Battle`、`Elite`、`Rest`、`Boss`

`EventProfile`
- 事件档位名。
- 运行时会根据这个值去 `EventOption.csv` 找对应的三选一配置。
- 当前已接入的档位：
  - `VillageStart`
  - `Rest`
  - `TradeMarket`
  - `InsightSanctum`
  - `DefaultBattle`
  - `DefaultElite`
  - `DefaultBoss`

`SpiritStoneElement`
- 当前关卡默认对应的五行灵石。
- 推荐填：`Metal`、`Wood`、`Water`、`Fire`、`Earth`
- 战斗掉落和部分事件默认会读取这个元素。

`ThemeZh`
- 当前关卡区域中文名。

`ThemeEn`
- 当前关卡区域英文名。

`DetailZh`
- 当前关卡节点说明中文。
- 会显示在地图节点详情里。

`DetailEn`
- 当前关卡节点说明英文。

`EventMode`
- 事件触发模式。
- 当前支持：`None`、`Fixed`、`Random`
- `Fixed` 代表固定服务点，可重复进入。
- `Random` 代表随机机缘点，触发后要等冷却恢复。

`CooldownMonths`
- 随机机缘冷却月数。
- 只有 `EventMode = Random` 时有意义。

`BattleExpReward`
- 当前关卡战斗胜利时的基础经验奖励。

`BattleSpiritStoneReward`
- 当前关卡战斗胜利时的基础灵石奖励。

`NonBattleExpReward`
- 当前关卡非战斗节点默认经验奖励。
- 主要给固定事件、随机机缘或休整节点的默认回报兜底使用。

`NonBattleSpiritStoneReward`
- 当前关卡非战斗节点默认灵石奖励。

`Notes`
- 备注。
- 只给开发和策划自己看，运行时不用。

### 使用规则

1. `Battle / Elite / Boss`
- 主要决定这个点是否进入战斗。
- 但如果 `EventProfile` 也绑定了特殊事件档位，仍会优先走事件逻辑。

2. `Rest`
- 代表这是一个非战斗节点。
- 实际出现什么选项，要看 `EventProfile` 对应哪组事件配置。

3. 改玩法位置时优先改这里
- 例如把第 4 关从 `TradeMarket` 改成普通休整：
  - `NodeType` 保持 `Rest`
  - `EventProfile` 改成 `Rest`

## EventOption.csv

### 作用

这张表定义某个事件档位下的三个选项内容。

例如：
- `TradeMarket` 可以有 3 行
- `OptionIndex = 1 / 2 / 3`
- 分别对应“换购法器 / 换购残卷 / 补足行囊”

### 字段说明

`Profile`
- 事件档位名。
- 必须和 `StageNode.csv` 的 `EventProfile` 对上。

`OptionIndex`
- 第几个选项。
- 当前约定每个 `Profile` 配 `1 / 2 / 3` 三个选项。

`TitleKey`
- 这个选项的标题多语言 key。
- 对应 `Localization.csv`。

`RewardMode`
- 奖励模式。
- 当前支持：
  - `Progress`
  - `Equipment`
  - `Skill`

`SpiritStoneElement`
- 使用哪种灵石元素。
- 当前推荐填：`Stage`
- `Stage` 的意思是“按当前关卡自动取对应五行灵石”。
- 也可以直接填具体元素，但当前主流程建议先统一用 `Stage`。

`ExpBase`
- 基础修为经验。

`ExpPerStage`
- 每关额外增加多少经验。
- 最终值公式：`ExpBase + Stage * ExpPerStage`

`SpiritStoneBase`
- 基础灵石奖励数量。

`SpiritStonePerStage`
- 每关额外增加多少灵石。
- 最终值公式：`SpiritStoneBase + Stage * SpiritStonePerStage`

`SpiritStoneCostBase`
- 基础灵石消耗。

`SpiritStoneCostPerStage`
- 每关额外增加多少灵石消耗。
- 最终值公式：`SpiritStoneCostBase + Stage * SpiritStoneCostPerStage`

`SkillRewardNodeType`
- 当 `RewardMode = Skill` 时生效。
- 决定这次功法机缘按哪种档位抽。
- 当前支持：
  - `Battle`
  - `Elite`
  - `Boss`

`SelectionTitleKey`
- 当 `RewardMode = Skill` 时，功法选择弹窗标题 key。

`SelectionMessageKey`
- 当 `RewardMode = Skill` 时，功法选择弹窗说明 key。

`ResultTitleKey`
- 这个选项结算结果窗标题 key。

`ResultIntroKey`
- 这个选项结算结果窗说明 key。

`EmptyResultKey`
- 当 `RewardMode = Skill` 且没有可用候选时显示的文案 key。
- 非 `Skill` 模式可以留空。

`Notes`
- 备注。
- 运行时不用。

## RewardMode 说明

### Progress
- 纯进度奖励。
- 会发修为经验、灵石。
- 不掉装备，不进功法三选一。

### Equipment
- 装备奖励型事件。
- 仍然可以同时给经验、灵石。
- 会额外走一次装备掉落逻辑。

### Skill
- 功法奖励型事件。
- 会进入一次功法三选一。
- 是否偏高阶，取决于 `SkillRewardNodeType`。

## 当前已接入的事件档位

`VillageStart`
- 新手村开局三选一。

`Rest`
- 普通休整三选一。

`TradeMarket`
- 资源交易点。
- 当前是“买法器 / 买残卷 / 补给”。

`InsightSanctum`
- 高阶机缘点。
- 当前是“凝练道基 / 开启遗藏 / 闭关顿悟”。

## 推荐改法

### 想改事件放在哪一关
改 `StageNode.csv`

例如：
- 第 4 关原本是 `TradeMarket`
- 想改到第 6 关

做法：
1. 把第 4 关 `EventProfile` 改掉
2. 把第 6 关 `EventProfile` 改成 `TradeMarket`

### 想改选项数值
改 `EventOption.csv`

例如：
- 想把 `TradeMarket` 第一个选项的灵石消耗从 `14 + 2*Stage` 改低
- 直接改：
  - `SpiritStoneCostBase`
  - `SpiritStoneCostPerStage`

### 想改功法档位
改 `SkillRewardNodeType`

例如：
- 把 `TradeMarket` 的残卷机缘从 `Elite` 调成 `Battle`
- 直接改 `SkillRewardNodeType`

## 当前约束

1. 每个 `Profile` 默认应有 3 个选项。
2. 多语言 key 必须在 `Localization.csv` 里存在。
3. `RewardMode = Skill` 时，建议把选择弹窗 key 和结果 key 都配齐。
4. 当前事件公式使用：
- `基础值 + 关卡 * 每关成长`

## 修改后要做什么

每次改完配置表后，在 Unity 里执行：
`工具/配置/导入全部CSV`

如果后面接了更完整的校验工具，也建议再跑一次：
`工具/配置/校验全部配置`
