# 策划交接文档

更新时间：2026-04-28

## 1. 用途

这份文档用于给新的策划/开发接手人快速说明：

- 当前版本已经做到哪
- 哪些文档是正式主文档
- 哪些系统已经接通
- 哪些内容还只是方案或试作
- 当前最值得继续推进的方向是什么

如果只看一份文档了解当前策划状态，优先看本文件。

## 2. 项目一句话

《五行行者》当前是一款竖屏、单机、修仙题材、地图推进 + 自动战斗 + 技能构筑 + 时间资源管理的肉鸽原型。

## 3. 当前版本状态

### 3.1 已经明确的主线方向

- 正式主线按 `120` 关骨架规划
- 结构为：
  - 金章 `1 - 20`
  - 木章 `21 - 40`
  - 水章 `41 - 60`
  - 火章 `61 - 80`
  - 土章 `81 - 100`
  - 圣章 `101 - 120`
- 五个章节 Boss 在：
  - `20 / 40 / 60 / 80 / 100`
- 最终圣系 Boss 在：
  - `120`

### 3.2 当前已接通的核心系统

- 地图推进
- 自动战斗
- 技能库 / 上场 3 主动技能 / 条件触发施法
- 装备实例与装备弹窗
- 事件系统
- 目标系统
- 剧情 / 对话 / 轻量分支
- 多语言
- 配置校验

### 3.3 当前策划上已定的重要口径

- 前台引导者正式称呼统一为：`问道系统`
- 剧情语气节奏采用：
  - 前期像系统文引导
  - 中期像观察者
  - 后期像裁定者
- 普通怪 / 精英怪保留五行标签
- 章节 Boss 机制风格对应五行，但元素关系应继续走配置，不应写死在代码里
- 装备方向采用：
  - `5` 品级：白 / 绿 / 蓝 / 紫 / 金
  - `5` 阶级：一阶 / 二阶 / 三阶 / 四阶 / 五阶
  - 当前版本暂不做强化

## 6. 剧情系统当前落地状态

### 6.1 已经正式接入的剧情配置链路

- `StoryNode.csv`
- `StoryTrigger.csv`
- `StoryChoice.csv`
- `Localization.csv`

### 6.2 当前真实状态

- 剧情系统已经不是纯单线，已支持轻量分支
- 已接入的能力包括：
  - 线性剧情
  - 对话
  - 选项
  - Flag / Value
  - 条件跳转
  - 按触发点播放
- 但六章正式剧情还没有整章完全落到配置表中
- 目前更接近：
  - 正式剧情主文档已完成
  - 配置侧只完成过开场和局部测试

### 6.3 接手建议

- 不要误以为剧情方案文档已经等于游戏内正式剧情完成
- 当前最合理的推进方式是：
  1. 先以 `章节剧情拆分方案.md` 为蓝本
  2. 逐章写 `StoryNode / StoryTrigger / StoryChoice`
  3. 最后补 `Localization.csv`

### 7.2 卡牌图当前约定

- 主角卡图建议按 `Character.csv` 里的角色 `ID` 命名
- 当前试作采用：
  - `200 x 300`
  - `PNG`
  - 透明底
- 小卡显示优先考虑半身像，不建议继续用全身立绘
- 我方卡图建议人物朝向画面右侧
- 敌方卡图后续建议人物朝向画面左侧

## 8. 当前最值得继续推进的方向

如果由新的策划或协作者接手，建议优先级如下：

### 第一优先级

- 把六章正式剧情按章落到 `StoryNode / StoryTrigger / StoryChoice / Localization`
- 同时统一 `问道系统` 的前中后期语气

### 第二优先级

- 继续完善卡牌角色图、敌方卡图和章节底框
- 让战斗页里的角色展示更稳定、更统一

### 第三优先级

- 继续扩地图章节视觉
- 继续补章节事件文案
- 配合目标系统做章节任务引导

## 9. 当前接手时要注意的坑

### 9.1 不要把文档方案当成已全部实装

正式剧情方案、章节拆分方案写得比较完整，但很多还没有完全进表。

### 9.2 美术试作和正式资源要分开看

`Assets/UI/Cards` 和 `Assets/UI/Frames` 里最近有试作资源，不一定都是最终版。

### 9.3 优先看“说明文档”和“配置表”，再看截图判断

这个项目最近迭代快，截图很容易过期。
接手时应优先看：

- `策划配置总说明.md`
- `策划给开发的文档.md`
- `正式剧情方案_五行试炼与圣道轮回.md`
- `章节剧情拆分方案.md`

### 9.4 文档编码需要留意

部分文档在不同终端或编码环境里可能出现乱码显示。
如果接手人本地看到 `章节剧情拆分方案.md` 或 `韩立工作总结.md` 乱码，优先检查：

- 文件编码是否为 UTF-8
- 当前终端/编辑器是否按 UTF-8 打开

## 10. 建议接手顺序

1. 先读本文件  
2. 再读：
   - `正式剧情方案_五行试炼与圣道轮回.md`
   - `章节剧情拆分方案.md`

   - `StoryNode.csv`

   - `Localization.csv`
4. 最后根据当前版本实际界面，决定下一步先推进剧情、数值还是美术资源

当前项目已经完成了“配置驱动的可玩原型”阶段，正式剧情和系统框架方向也已经定住；下一位接手者最应该做的，不是重新发散方向，而是**沿着既定主线，把六章剧情、美术卡图和章节表达逐步落到配置与资源里**。
