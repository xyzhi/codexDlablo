# 《五行行者》单机肉鸽游戏策划大纲

## 一、项目定位

- 类型：单机修仙题材肉鸽游戏
- 核心循环：地图推进 -> 节点事件/战斗 -> 战后成长 -> 继续推进
- 战斗形式：自动战斗为主，玩家通过构筑、装备、路线与资源管理影响结果
- 题材关键词：五行、灵根、功法、寿元、坐化、破境

## 二、核心体验目标

1. 让玩家感受到“修仙途中与寿命赛跑”的压力。
2. 让战斗、地图、成长形成完整单局闭环。
3. 让五行构筑、装备调整、路线取舍彼此联动。
4. 第一版先追求可玩和可迭代，不追求重美术。

## 三、基础玩法结构

### 1. 角色与构筑

- 玩家以固定小队进入单局。
- 角色拥有五行灵根、基础属性、初始功法、定位差异。
- 多灵根角色可接触更广功法池，但单一功法成长上限更低。
- 装备与功法共同决定角色的单局流派方向。

### 2. 战斗

- 战斗以自动战斗为主。
- 玩家在战斗外通过装备、功法、路线、资源做决策。
- 战斗内重点反馈：日志、血量、法力、技能释放、死亡、胜负结算。
- 第一版以稳定、快速、可调试为优先。

### 3. 单局成长

- 战后可获得功法、装备或其他局内收益。
- 地图节点推进时逐步提高风险与奖励。
- 单局中不做复杂局外养成，重点先放在单局闭环。

## 四、地图推进方案

### 1. 第一版地图结构

- 第一版明确采用一维线性节点地图。
- 节点类型先做四类：
- `Battle`
- `Elite`
- `Rest`
- `Boss`
- 玩家在第一版中沿单一路线推进，重点验证寿元消耗、节点节奏、战斗回流与单局闭环。

### 2. 地图循环

地图主流程为：

`主菜单 -> 地图 -> 节点选择 -> 战斗/事件 -> 结算 -> 返回地图 -> 继续推进`

### 3. 区域章节

地图按区域推进，建议章节名如下：

1. 山门外围
2. 焚风古道
3. 地宫遗迹
4. 登天阶

### 4. 地图设计目标

- 第一版先保证单局流程顺畅，采用一维地图。
- 当副本系统完成后，再升级为二维地图或纵向分叉节点图。
- 二维化后的重点是强化路线选择、事件分布与资源取舍，而不是一开始就增加复杂度。

## 五、寿元纪年方案

### 1. 设计目标

- 寿命不是背景装饰，而是核心资源。
- 玩家每前进一步都在消耗寿元。
- 寿元压力与战斗、路线、奖励形成取舍。

### 2. 时间推进规则

- 地图上每前进一步，默认增加一月。
- 普通战斗、休息点、商店：默认耗时一月。
- 精英战、Boss 战、特殊事件：可提高到两月。
- 特殊事件后续可扩展为 0 到数月的波动消耗。

### 3. 寿元显示形式

采用第三种寿元纪年显示方案：

- `阳寿：余二十六载七月`

推荐统一格式：

- `阳寿：余X载X月`

该方案优点：

- 古风味足
- 可读性强
- 玩家能直观看懂剩余寿命

### 4. 地图中的时间提示

节点推进时可弹出短提示：

- `耗时一月`
- `又逝一月`
- `闭关半载`
- `鏖战两月`

### 5. 寿终失败条件

- 当累计时间达到主角寿命上限时，本轮立即结束。
- 结算文案建议：

`阳寿已尽。主角未能再进一步，坐化于修行途中。`

### 6. 后续可扩展方向

- 灵果增寿
- 禁术燃命
- 破境返寿
- 奇遇换寿元

这样寿命系统就能从“倒计时”升级为“可经营资源”。

## 六、战斗与地图的联动关系

- 路线越贪，收益越高，但寿元消耗越大。
- 精英节点更危险，但给更好装备或功法。
- 休息点可以稳住状态，但也会消耗寿元。
- 如果战斗预估不理想，玩家应优先调整装备再开战。
- 如果寿元紧张，玩家需要放弃部分收益路线。

## 七、第一版推荐落地方向

当前推荐组合：

- 一维线性地图
- 每步一月
- 寿元纪年显示
- 寿尽即本轮结束
- 自动战斗 + 局外决策驱动

后续在副本系统稳定后，再演进为二维地图版本。

## 八、当前开发优先级

1. 打通主菜单 -> 地图 -> 战斗 -> 结算 -> 返回地图
2. 接入寿元推进与寿终结算
3. 接入节点类型差异
4. 接入战后奖励
5. 副本系统完成后，再升级到二维地图
