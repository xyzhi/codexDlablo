# 策划配置总说明

这份文档以后作为开发写给策划的固定说明文件使用。

用途是：
- 告诉策划当前哪些表已经正式接入
- 说明各字段在运行时里的实际含义与约束
- 说明当前版本前提、配表边界、校验方式

如果是策划要提给开发的新需求、希望补的系统能力、配置化诉求，不写在这份里，统一写到：
`Docs/策划给开发的文档.md`

## 当前版本前提

- 我方默认只有 1 名角色上阵：`C001 金锋卫`
- 功法有 `品质 + 等级`
- 重复获得同功法，不是重复进背包，而是该功法 `+1 级`
- 普通 / 精英 / Boss 的功法奖励档位不同
- 地图总关卡数按 `StageNode.csv` 读取
- Boss 关卡位置按 `StageNode.csv` 配置，不再固定写死
- 最终首领关由策划在 `StageNode.csv` 中配置
- 固定事件可以反复进入，但不应该无成本无限刷收益
- 随机机缘是冷却制，不是一次性永久消失
- 普通怪 / 精英怪保留五行标签与五行特征
- 章节 Boss 统一使用 `圣元素标签`
- `圣元素` 不参与五行增伤、减伤、克制判定，但 Boss 机制风格仍分别对应 `金 / 木 / 水 / 火 / 土`

## 配置优先约定

- 数值、文案、章节、节点、事件、奖励、元素关系，尽可能都走配置。
- 策划主维护 `Docs/*.csv`。
- 运行时主读取 `Assets/Resources/Configs/*.json` 与 `Assets/Resources/Localization/GameText.json`。
- 新系统如果存在长期调优需求，优先做成 `csv/json` 可调，不要先写死在代码里。
- 代码里保留默认值可以，但最终生效逻辑应优先读配置。

## StageNode.csv

这张表决定：
1. 每一关是什么节点
2. 这一关走哪组事件档位
3. 这一关默认掉什么五行灵石
4. 这一关基础战斗 / 非战斗奖励是多少

### 字段说明

`Stage`
- 关卡编号

`NodeType`
- 节点类型
- 当前支持：`Battle`、`Elite`、`Rest`、`Boss`

`EventProfile`
- 事件档位名
- 运行时会根据这个值去 `EventOption.csv` 取这组玩法的 3 个选项

`SpiritStoneElement`
- 当前关卡默认对应的五行灵石
- 推荐填：`Metal`、`Wood`、`Water`、`Fire`、`Earth`

`ThemeZh` / `ThemeEn`
- 关卡区域名

`DetailZh` / `DetailEn`
- 地图节点详情说明

`EventMode`
- 事件模式
- 当前支持：`None`、`Fixed`、`Random`
- `Fixed`：固定服务点，可重复进入
- `Random`：随机机缘点，触发后要等冷却恢复

`CooldownMonths`
- 随机机缘冷却月数
- 只有 `EventMode = Random` 时有意义

`BattleExpReward`
- 当前关卡战斗胜利时的基础经验奖励

`BattleSpiritStoneReward`
- 当前关卡战斗胜利时的基础灵石奖励

`NonBattleExpReward`
- 当前关卡非战斗节点默认经验奖励

`NonBattleSpiritStoneReward`
- 当前关卡非战斗节点默认灵石奖励

`Notes`
- 备注，不参与运行时

### 推荐用途

- 想改某关是什么玩法：改这张表
- 想把商店、传送、随机机缘挪到别的关：改这张表
- 想调某关基础经验和灵石：改这张表

## EnemyEncounter.csv

这张表决定战斗节点实际刷哪组敌人。

### 字段说明

`ID`
- 编组配置 ID

`StageFrom` / `StageTo`
- 生效关卡范围

`NodeType`
- 生效节点类型
- 当前支持：`Battle`、`Elite`、`Boss`

`EnemyIds`
- 这一编组实际出场的敌人 ID
- 多个敌人用 `|` 分隔

`OverrideElement`
- 可选
- 填了以后，这组敌人的战斗元素会统一覆写
- 当前主要给 Boss 填 `Holy`

`Weight`
- 当同一关卡范围、同一节点类型配了多组编组时，按权重随机

`Notes`
- 备注，不参与运行时

### 推荐用途

- 想改某章普通战刷什么怪：改这张表
- 想改某章精英战混编什么怪：改这张表
- 想改 Boss 用圣元素还是别的特殊标签：改这张表

## EventOption.csv

这张表定义某个事件档位下的 3 个选项内容。

### 字段说明

`Profile`
- 事件档位名
- 必须和 `StageNode.csv` 的 `EventProfile` 对上

`OptionIndex`
- 第几个选项
- 当前约定每组配 `1 / 2 / 3`

`TitleKey`
- 选项标题多语言 key

`RewardMode`
- 奖励模式
- 当前支持：`Progress`、`Equipment`、`Skill`、`Utility`

`UtilityAction`
- 当 `RewardMode = Utility` 时生效
- 当前用于打开装备整理、功法总览、路线说明这类服务动作

`SpiritStoneElement`
- 使用哪种灵石元素
- 推荐填：`Stage`
- `Stage` 的意思是“按当前关卡自动取对应五行灵石”

`ExpBase` / `ExpPerStage`
- 经验公式：`ExpBase + Stage * ExpPerStage`

`SpiritStoneBase` / `SpiritStonePerStage`
- 灵石奖励公式：`SpiritStoneBase + Stage * SpiritStonePerStage`

`SpiritStoneCostBase` / `SpiritStoneCostPerStage`
- 灵石消耗公式：`SpiritStoneCostBase + Stage * SpiritStoneCostPerStage`

`SkillRewardNodeType`
- 当 `RewardMode = Skill` 时生效
- 当前支持：`Battle`、`Elite`、`Boss`

`SelectionTitleKey` / `SelectionMessageKey`
- 功法选择弹窗文案 key

`ResultTitleKey` / `ResultIntroKey`
- 结果弹窗文案 key

`EmptyResultKey`
- 没有可用功法候选时显示的文案 key

`Notes`
- 备注，不参与运行时

## Skill.csv

策划当前主要在这张表里负责：
- 功法品质
- 基础强度
- 消耗
- 持续时间
- 描述文案
- 哪些技能适合升级成长

### 当前规则

- 功法品质分档：`普通`、`稀有`、`绝品`

- 精英更偏高品质
- Boss 更偏顶品质

## Skill.csv

这张表负责技能本体信息：
- 技能名称
- 五行
- 品质
- 类型
- 目标类型
- MP 消耗
- 展示描述

### 当前规则

- `Skill.csv` 负责“这个技能是什么”
- `SkillEffect.csv` 负责“这个技能实际产生哪些效果”
- 一个技能可以挂多个效果，例如：
  - 先造成伤害
  - 再附加破甲
  - 或者先套盾再持续回血

## StoryNode.csv

这张表负责“剧情节点本体”，支持：
- 全屏剧情 `Story`
- 盖在任意界面上的对话 `Dialog`

### 字段说明

`Id`
- 剧情节点 ID

`Type`
- 节点类型
- 当前支持：`Story`、`Dialog`

`SpeakerKey`
- 旧兼容字段
- 如果只配这一项，对话会默认把它当作左侧说话人

`LeftSpeakerKey`
- 左侧说话人名字的多语言 key

`RightSpeakerKey`
- 右侧说话人名字的多语言 key

`ActiveSpeakerSide`
- 当前这句是谁在说话
- 当前支持：`Left`、`Right`

`TitleKey`
- 剧情标题 key
- 主要给 `Story` 用

`ContentKey`
- 正文内容 key

`TypingCharsPerSecond`
- 打字机速度

`SkipHintDelay`
- 过多久后显示“点击任意位置继续/跳过”

`Skippable`
- 是否允许点击跳过打字过程

`NextNodeId`
- 下一句跳转到哪个剧情节点
- 可以 `Story -> Dialog`
- 也可以 `Dialog -> Story`

`CallbackKey` / `CallbackParam`
- 节点结束时回调给程序的 key 和参数
- 用于在关键节点触发开战、结算、引导等

`Notes`
- 备注，不参与运行时

### 推荐用途

- 开场剧情
- 战斗前对白
- 战斗后对白
- 关键节点引导
- 章节结尾演出

### 配置建议

- 纯旁白用 `Story`
- 需要左右说话人感觉的用 `Dialog`
- 对话建议显式填写：
  - `LeftSpeakerKey`
  - `RightSpeakerKey`
  - `ActiveSpeakerSide`
- 如果是双人对话，连续两句分别切 `ActiveSpeakerSide = Left / Right`

## StoryTrigger.csv

这张表负责“什么时候触发剧情/对话”。

### 字段说明

`Id`
- 触发器 ID

`TriggerKey`
- 触发点名
- 例如：`StartGame`、`BeforeBattle`、`AfterBattle`

`Stage`
- 对应关卡
- 同一个触发点可以按关卡区分不同内容

`NodeId`
- 触发后进入的首个剧情节点 ID

`OncePerRun`
- 本轮是否只触发一次

`Enabled`
- 是否启用

`Notes`
- 备注，不参与运行时

### 当前已接入的示例

- `StartGame`
  - 点击开始游戏时触发

### 运行规则

- 剧情/对话触发时会暂停时间机制
- 整个链条完全结束后再恢复
- 支持连续跳转多个节点
- 最后一个节点结束后才执行回调

## 本地化约定

- 剧情和对话文案统一写在 `Docs/Localization.csv`
- 说话人名字也走多语言 key
- 新增剧情节点时，至少补这些 key：
  - 标题 key（如果有）
  - 正文 key
  - 左右说话人 key（如果有）
- 对应 `Skill.csv` 里的技能 ID

`EffectIndex`
- 效果顺序
- 同一个技能可以配多行，运行时会按顺序执行

`EffectType`
- 当前支持：`Damage`、`Heal`、`Shield`、`Dot`、`Hot`、`Control`、`ArmorBreak`、`Vulnerable`、`ManaGain`、`AttackUp`、`DefenseUp`、`DotBoost`

`TargetScope`
- 当前支持：`Self`、`SingleEnemy`、`SingleAlly`、`AllEnemies`、`AllAllies`、`Default`

`Value`
- 基础数值

`ValuePerLevel`
- 每升 1 级额外增加的数值

`DurationRounds`
- 持续回合数
- 即时效果填 `0`

`MaxStacks`
- 最大叠层数

`StackRule`
- 当前支持：`Refresh`、`Stack`
- `Refresh`：刷新持续时间
- `Stack`：叠层到上限

`TriggerTiming`
- 当前支持：`Instant`、`TurnStart`、`TurnEnd`、`Duration`、`Passive`、`Control`

`Notes`
- 备注，不参与运行时

### 当前使用建议

- 直接伤害：`Damage`
- 持续伤害：`Dot`
- 持续治疗：`Hot`
- 护盾：`Shield`
- 控制一回合：`Control`
- 破甲：`ArmorBreak`
- DOT 被动增幅：`DotBoost`

## 固定事件与随机机缘规则

### 固定事件
- 可以反复进入
- 本质是服务点、交易点、整理点
- 不应该设计成无成本无限刷纯收益

### 随机机缘
- 会随机触发一个事件结果
- 触发后进入冷却
- 过若干月后恢复
- 更适合配 buff、掉落、特殊奇遇

## 你现在优先可以改什么

1. `StageNode.csv`
- 调整事件点放在哪一关
- 调整每关基础经验和灵石
- 调整每关五行灵石类型

2. `EventOption.csv`
- 调整事件三选一的数值、消耗、收益
- 调整商店 / 静室 / 奇遇 / 熔炉的强度节奏

3. `Skill.csv`
- 调整功法品质、威力、费用、持续时间
- 调整重复升级后的强度曲线

4. `Localization.csv`
- 修改配置 key 对应的正式文案

## 当前建议重点验证

- 单主角开局第 `2 / 3 / 4` 关曲线
- 当前配置下各个 Boss 关的章节压力是否合理
- 五行普通怪 / 精英怪特征是否鲜明
- 圣元素 Boss 是否保留风格，但不会把某个主角克死
- 固定事件是否存在无成本刷收益
- 随机机缘冷却是否符合预期
- 同功法连续升 `2~3` 级后是否失衡
- 精英 / Boss 奖励品质是否有明显抬升
- 中后期事件收益能否跟上 `100` 关节奏
