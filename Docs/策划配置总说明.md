# 策划配置总说明

这份文档以后作为策划唯一固定说明文件使用。

当前主要改这几张表：
- `Docs/StageNode.csv`
- `Docs/EventOption.csv`
- `Docs/EventProfile.csv`
- `Docs/Skill.csv`
- `Docs/Localization.csv`

## 当前版本前提

- 我方默认只有 1 名角色上阵：`C001 金锋卫`
- 功法有 `品质 + 等级`
- 重复获得同功法，不是重复进背包，而是该功法 `+1 级`
- 普通 / 精英 / Boss 的功法奖励档位不同
- 地图现在是 `100` 关，`5` 个 Boss
- 固定事件可以反复进入，但不应该无成本无限刷收益
- 随机机缘是冷却制，不是一次性永久消失

## StageNode.csv

这张表决定：
1. 每一关是什么节点
2. 这一关走哪组事件档位
3. 这一关默认掉什么五行灵石
4. 这一关基础战斗 / 非战斗奖励是多少

### 字段说明

`Stage`
- 关卡编号

`NodeType`
- 节点类型
- 当前支持：`Battle`、`Elite`、`Rest`、`Boss`

`EventProfile`
- 事件档位名
- 运行时会根据这个值去 `EventOption.csv` 取这组玩法的 3 个选项

`SpiritStoneElement`
- 当前关卡默认对应的五行灵石
- 推荐填：`Metal`、`Wood`、`Water`、`Fire`、`Earth`

`ThemeZh` / `ThemeEn`
- 关卡区域名

`DetailZh` / `DetailEn`
- 地图节点详情说明

`EventMode`
- 事件模式
- 当前支持：`None`、`Fixed`、`Random`
- `Fixed`：固定服务点，可重复进入
- `Random`：随机机缘点，触发后要等冷却恢复

`CooldownMonths`
- 随机机缘冷却月数
- 只有 `EventMode = Random` 时有意义

`BattleExpReward`
- 当前关卡战斗胜利时的基础经验奖励

`BattleSpiritStoneReward`
- 当前关卡战斗胜利时的基础灵石奖励

`NonBattleExpReward`
- 当前关卡非战斗节点默认经验奖励

`NonBattleSpiritStoneReward`
- 当前关卡非战斗节点默认灵石奖励

`Notes`
- 备注，不参与运行时

### 推荐用途

- 想改某关是什么玩法：改这张表
- 想把商店、传送、随机机缘挪到别的关：改这张表
- 想调某关基础经验和灵石：改这张表

## EventOption.csv

这张表定义某个事件档位下的 3 个选项内容。

### 字段说明

`Profile`
- 事件档位名
- 必须和 `StageNode.csv` 的 `EventProfile` 对上

`OptionIndex`
- 第几个选项
- 当前约定每组配 `1 / 2 / 3`

`TitleKey`
- 选项标题多语言 key

`RewardMode`
- 奖励模式
- 当前支持：`Progress`、`Equipment`、`Skill`、`Utility`

`UtilityAction`
- 当 `RewardMode = Utility` 时生效
- 当前用于打开装备整理、功法总览、路线说明这类服务动作

`SpiritStoneElement`
- 使用哪种灵石元素
- 推荐填：`Stage`
- `Stage` 的意思是“按当前关卡自动取对应五行灵石”

`ExpBase` / `ExpPerStage`
- 经验公式：`ExpBase + Stage * ExpPerStage`

`SpiritStoneBase` / `SpiritStonePerStage`
- 灵石奖励公式：`SpiritStoneBase + Stage * SpiritStonePerStage`

`SpiritStoneCostBase` / `SpiritStoneCostPerStage`
- 灵石消耗公式：`SpiritStoneCostBase + Stage * SpiritStoneCostPerStage`

`SkillRewardNodeType`
- 当 `RewardMode = Skill` 时生效
- 当前支持：`Battle`、`Elite`、`Boss`

`SelectionTitleKey` / `SelectionMessageKey`
- 功法选择弹窗文案 key

`ResultTitleKey` / `ResultIntroKey`
- 结果弹窗文案 key

`EmptyResultKey`
- 没有可用功法候选时显示的文案 key

`Notes`
- 备注，不参与运行时

## EventProfile.csv

这张表只负责事件档位本身的标题和主说明。

### 字段说明

`Profile`
- 事件档位名

`TitleKey`
- 事件标题 key

`MessageKey`
- 事件主说明 key

`Notes`
- 备注，不参与运行时

## Skill.csv

策划当前主要在这张表里负责：
- 功法品质
- 基础强度
- 消耗
- 持续时间
- 描述文案
- 哪些技能适合升级成长

### 当前规则

- 功法品质分档：`普通`、`稀有`、`绝品`
- 重复拿到同功法会 `+1 级`
- 精英更偏高品质
- Boss 更偏顶品质

## 固定事件与随机机缘规则

### 固定事件
- 可以反复进入
- 本质是服务点、交易点、整理点
- 不应该设计成无成本无限刷纯收益

### 随机机缘
- 会随机触发一个事件结果
- 触发后进入冷却
- 过若干月后恢复
- 更适合配 buff、掉落、特殊奇遇

## 你现在优先可以改什么

1. `StageNode.csv`
- 调整事件点放在哪一关
- 调整每关基础经验和灵石
- 调整每关五行灵石类型

2. `EventOption.csv`
- 调整事件三选一的数值、消耗、收益
- 调整商店 / 静室 / 奇遇 / 熔炉的强度节奏

3. `Skill.csv`
- 调整功法品质、威力、费用、持续时间
- 调整重复升级后的强度曲线

4. `Localization.csv`
- 修改配置 key 对应的正式文案

## 当前建议重点验证

- 单主角开局第 `2 / 3 / 4` 关曲线
- 固定事件是否存在无成本刷收益
- 随机机缘冷却是否符合预期
- 同功法连续升 `2~3` 级后是否失衡
- 精英 / Boss 奖励品质是否有明显抬升

## 改表后的操作

每次改完后，在 Unity 里执行：
`工具/配置/导入全部CSV`

如果配置校验工具已接好，再执行：
`工具/配置/校验全部配置`
