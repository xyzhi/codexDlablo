# 配置校验工具说明

## 入口
在 Unity 菜单中使用：`工具/配置/校验全部配置`

## 当前支持的表
- `Docs/Character.csv`
- `Docs/Enemy.csv`
- `Docs/Skill.csv`
- `Docs/Equipment.csv`
- `Docs/SpiritStone.csv`
- `Docs/StageBalance.csv`
- `Docs/StageNode.csv`
- `Docs/EventOption.csv`
- `Docs/Localization.csv`

## 主要会检查什么
### 通用类
- 重复 ID / 重复 key
- 必填字段为空
- 数值列无法解析
- 疑似乱码字符

### 技能与角色类
- 技能品质是否合法
- 五行字段是否合法
- 角色初始技能、敌人技能引用是否存在

### 关卡成长表
- `Stage` 是否重复
- `Stage` 是否连续
- `EnemyEquipmentCount` 是否超常规范围
- 明显异常数值会给 warning

### 关卡节点表
- `Stage` 是否重复或断档
- `NodeType` 是否为 `Battle / Elite / Rest / Boss`
- `EventProfile` 是否为空或引用不存在
- 中英文主题与说明是否缺失

### 事件选项表
- `Profile + OptionIndex` 是否重复
- 每个 `Profile` 是否完整拥有 3 个选项
- `RewardMode` 是否为 `Progress / Equipment / Skill`
- `SkillRewardNodeType` 是否为 `Battle / Elite / Boss` 或留空
- 成本和奖励数值是否为负数
- 多语言 key 是否为空或不存在

### 语言表
- key 是否重复
- 中文 / 英文是否为空
- 多行文本换行是否异常
- 疑似乱码 warning

## 日志怎么看
### 错误
- 以 `[错误][表名]` 开头
- 一般表示会直接影响运行、导入或配置引用
- 需要优先修改

### 警告
- 以 `[警告][表名]` 开头
- 一般表示高风险配置、疑似异常值或可疑文案
- 不一定阻塞运行，但建议尽快确认

## 常见问题对应改哪一列
- 技能引用无效：去改 `Character.csv` 的 `InitialSkills` 或 `Enemy.csv` 的 `Skills`
- 节点类型不合法：去改 `StageNode.csv` 的 `NodeType`
- 事件配置不完整：去改 `EventOption.csv` 的 `Profile`、`OptionIndex`
- 多语言 key 不存在：去改 `Localization.csv` 的 `Key`，或修正配置里引用的 key
- 灵石颜色不一致：去改 `SpiritStone.csv` 的 `ColorHex`

## 建议用法
1. 策划改完 CSV 后先跑一次校验
2. 先清错误，再看 warning
3. 如果是事件或节点新增，优先一起检查 `StageNode.csv`、`EventOption.csv`、`Localization.csv`
